160  static int findRegion(EGS_Float xp, int np, const EGS_Float *p) {
161  int ml = 0, mu = np;
162  while (mu - ml > 1) {
163  int mav = (ml+mu)/2;
164  if (xp <= p[mav]) {
165  mu = mav;
166  }
167  else {
168  ml = mav;
169  }
170  }
171  return mu - 1;

1121 /* \example geometry/example1/geometry_example1.cpp
1122 
1123  Suppose that you frequently use the same complex geometry and you
1124  are tired of always having to include the long definition of this geometry
1125  into your input file. In this case it may be worthwhile to implement your
1126  own geometry DSO that constructs the geometry of interest with very little
1127  input (or even no input at all). This example illustrates how to do this
1128  programatically.
1129 
1130  As this is just an example, the geometry we will be constructing is not very
1131  complex: it consists of a cylinder inscribed in a sphere and the sphere
1132  inscribed in a box as shown in the figure ???. We want to be able to define
1133  the media of the cylinder, the sphere and the box in the input file
1134  (so that we can easily use different PEGS data sets, for example). All other
1135  dimensions of our example geometry are fixed.
1136 
1137  To create our new geometry, we make a new subdirectory in the egspp
1138  geometries subdirectory called \c example1. To be able to use the
1139  predefined rules for building geometry libraries we create a dummy
1140  header file \c %geometry_example1.h that does nothing else except to
1141  define the \c EGS_EXAMPLE1_DLL
1142  macro:
1143  \include geometry/example1/geometry_example1.h
1144  and put the implementation in \c %geometry_example1.cpp.
1145  \dontinclude geometry/example1/geometry_example1.cpp
1146  In this file we include the just created header file and the EGS_Input
1147  class header file to get access to its definition,
1148  needed in the \c %createGeometry() function (see below):
1149  \until egs_input
1150  We also need the declarations of the various geometry classes that we
1151  will use to construct our geometry:
1152  \until egs_envelope_geometry.h
1153  We then define the dimensions of the various geometrical structures
1154  \until box_size
1155 
1156  The remaining task is to implement a C-style \c %createGeometry() function
1157  exported by the DSO
1158  \until EGS_EXAMPLE1_EXPORT
1159  We check that the input object is valid and return \c null if it is not
1160  \until }
1161  We then look for a definition of the cylinder medium and print a warning if
1162  such a definition is missing:
1163  \skipline cyl_medium
1164  \until }
1165  We repeat basically the same code to obtain the sphere and the box
1166  media
1167  \until }
1168  \until }
1169  and return \c null if any of the media definition was missing:
1170  \until if
1171  Now we are ready to construct our geometry. We first create a set of
1172  parallel z-planes
1173  \until EGS_ZProjector
1174  As the planes will only be used by our geometry object internally, we don't
1175  care how the plane set is named and therefore give it a name using
1176  the static function EGS_BaseGeometry::getUniqueName().
1177  In a similar way we create a set of z-cylinders consisting of a single
1178  cylinder
1179  \until ZProjector
1180  and make a closed cylinder modeled as a 2D-geometry from the planes and the
1181  cylinder surface:
1182  \until getUniqueName
1183  We then set the medium of the cylinder
1184  \until setMedium
1185  The next step is to create the sphere as a set of spheres consisting of
1186  a single sphere
1187  \until getUniqueName
1188  and to fill it with the sphere medium defined in the input
1189  \until setMedium
1190  We then use an envelope geometry to inscribe the cylinder into the just
1191  created sphere:
1192  \until getUniqueName
1193  The next step is to create the outer box and fill it with the box medium
1194  defined in the input
1195  \until setMedium
1196  (as our geometry will always be centered about the origin, we pass a
1197  \c null affine transformation to the constructor of the box).
1198  We now inscribe the \c the_sphere geometry object (which is the sphere
1199  with the cylinder inscribed in it) into the just created box using
1200  again an envelope geometry,
1201  \until the_box
1202  set the name of \c the_box from the input using the inherited
1203  \link EGS_BaseGeometry::setName(EGS_Input*) setName() \endlink method,
1204  \until setName
1205  and return a pointer to the just created geometry
1206  \until }
1207  \until }
1208  That's all.
1209 
1210  In the Makefile we include the egspp make config files,
1211  \dontinclude geometry/example1/Makefile
1212  \skipline EGS_CONFIG
1213  \until my_machine
1214  set the C-preprocessor defines to the standard set of defines
1215  for our egspp configuration,
1216  \until DEFS
1217  define the name of our DSO,
1218  \until library
1219  and and the set of files needed to build the DSO
1220  \until lib_files
1221  Because we are using planes, cylinders, spheres, a ND-geometry, a box
1222  geometry and an envelope geometry to construct our geometry, we must
1223  link against the geometry libraries containing these geometries.
1224  We accomplish this by seting the \c link2_libs make variable to the
1225  list of needed libraries
1226  \until egs_genvelope
1227  We then include the standard rules for building a DSO
1228  \until include
1229  and set the dependencies of our object files
1230  \until make_depend
1231  We must also add the dependencies on the various geometry header files
1232  that we are including
1233  \until egs_envelope_geometry.h
1234  We can now build our geometry DSO by typing \c make and use
1235  it by including input like this in the input file
1236  \verbatim
1237  :start geometry:
1238  library = geometry_example1
1239  name = some_name
1240  cylinder medium = H2O521ICRU
1241  sphere medium = AL521ICRU
1242  box medium = AIR521ICRU
1243  :stop geometry:
1244  \endverbatim
1245  It is worth noting that we could have constructed this geometry as the
1246  \link EGS_UnionGeometry union \endlink of the cylinder pointed to
1247  by \c the_cyl, the sphere pointed to by \c sphere and the
1248  box pointed to by \c box. However, such an implementation would be
1249  much slower at run time because for each invocation of a geometry
1250  method the union will have to interogate the geometry methods of all
1251  3 objects. In contrast, the envelope geometry will be checking only
1252  against the box for points outside of the box, the box and the sphere for
1253  points inside the box but outside the sphere, the sphere and the cylinder
1254  for points inside the sphere but outside the cylinder and the cylinder
1255  only for points inside the cylinder.
1256 
1257  The complete source code of this example geometry is below
1258  \include geometry/example1/Makefile
1259  \include geometry/example1/geometry_example1.h
1260
